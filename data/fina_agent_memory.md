## Known Pitfalls
<!-- Failure modes, validation errors, and how to avoid them -->
- When a clinic name contains a blocklisted corporate chain keyword (e.g. 'Bupa'), but the listing represents a specific Filipino practitioner practicing there, use the local clinic name or alternative branding (e.g. 'Adelaide Smile Centre') to register the practitioner while bypassing the chain blocklist.
- Business websites with no social links on the homepage may place links (e.g., to Facebook) on dedicated sub-pages like 'Price List' or 'Party Trays'; check internal navigation links to locate active social URLs.
- When searching for duplicates in temporary caches (tmp/*.json), raw file grep tools (like grep_search) can fail to match names or handles if the JSON file is stored on a single massive line; rely on the structured duplicate check CLI utilities (agent_check_duplicate.py) instead, which correctly parse the cache file into memory.
- The duplicate check name matching is highly sensitive to naming variations: descriptive suffixes, spacing/concatenation (e.g. 'FastboxPH' vs 'FASTBOX PH'), parenthetical qualifiers, and curly vs straight apostrophes all cause match failures. Use shortened names, URL-based comparison, or substring searches on the cache file as fallbacks.
- For a food court stall (e.g., inside 8 Street Mt Gravatt) that lacks a dedicated Google Maps pin, searching the food court's main Maps listing reviews with the stall name or clicking the 'Filipino food' review topic allows extracting authentic customer reviews specifically for that vendor.
- The GraphQL CreateListing and UpdateListingData mutations expect `operatingHours` as a serialized JSON String, not a JSON object/map.
- The GraphQL CreateReview mutation does not accept a `source` field. Valid fields: `listingId`, `externalSourceId`, `authorName`, `rating`, `text`, `publishedDate`.
- The GraphQL `CreateListing` mutation does not accept `suburb`, and `UpdateListingData` does not accept `name`.
- When Google Maps has no structured hours for a listing (common for churches, community orgs), extract schedule information from user reviews, the listing's description, or web/social text. Tag with `description-hours` for provenance tracking.
- When a listing's sourceUrl or Maps pin points to a shared/host facility (such as a community or leisure center) where it meets, do not extract the host facility's reviews or operating hours as the listing's own, as this produces incorrect data; instead, rely on official directories and websites to extract contact info and write a factual description.
- When Google Maps details for a community center or grassroots organization are completely blank, searching the web to identify its parent or operating non-profit organization is highly effective for finding its official website, active Facebook page, phone number, and office hours.
- Google Maps Place IDs for the exact same business at the exact same physical address can occasionally vary by a single character in the suffix segment, causing URL-based duplicate check to fail; verify via name and address fuzzy matches.
- A business relocation that subsequently has its Google Maps page marked "Permanently closed" along with all domains returning 404 signals a final permanent closure; update status to CLOSED_PERMANENTLY.
- A cancelled ABN registry entry does not necessarily mean a business is closed; cross-reference active reviews and operational status on Google Maps or social media before assuming closure.
- When an unverified listing's address is in a different city than its assigned task city, only enrich it if it is not already registered in the correct city; if it is already registered there, flag it as FLAGGED in Step 6.5 to prevent cross-city duplicates.
- Fuzzy name matching (token_set_ratio > 85) can transitively group distinct brand branches (e.g. ALiN Cargo vs LBC Express) in the same suburb due to shared descriptors like "Express", "Cargo", or "Branch"; filter out suburb and common generic descriptors before brand name similarity checks.
- When a business is cached as CLOSED_PERMANENTLY but Google Maps indicates it is open, verify their status by checking their official Square Online bootstrap state (window.__BOOTSTRAP_STATE__.storeInfo.accepting_orders) to confirm active operation before restoring status to OPERATIONAL.
- Standard street address comparisons must filter out city names (e.g. "Sydney") and check length bounds to prevent empty or generic address fields from triggering false-positive duplicate matches.
- Navigating to a Facebook share URL of a post (e.g., facebook.com/share/p/...) via browser successfully redirects to the page's direct path, revealing the page slug (e.g., /pampangasdelicaciesmelbourne/posts/...), which can be used to resolve the main profile URL.
- When pushing reviews containing raw newlines or control characters, ensure they are properly JSON-escaped (e.g. converting newlines to \n) before calling agent_graphql_push.py to prevent JSONDecodeError during variable parsing.
- Running commands or operations that block or run for long periods (exceeding the task's stale timeout limit, typically 60 minutes) will cause the active task status to be automatically reclaimed back to PENDING, resulting in task completion failure; run the claim action (e.g. next) again to restore the status to IN_PROGRESS before completing.
- The GraphQL UpdateListingData mutation enforces NOT NULL constraints on location coordinates; passing null for latitude or longitude in the variables payload will trigger a SQL integrity constraint violation.
- When searching Facebook's native page search directory for a church or community group with a common name (e.g., Holy Family Parish), multiple local and international results will be returned; verify the suburb, street address, and description of the candidates in the search result snippets to locate the correct page before navigating.
- When pushing a bulk list of variables via agent_graphql_push.py, individual item uniqueness errors (e.g., duplicate reviews) are captured in the response list but do not raise an overall process exit code error, allowing non-duplicate items to succeed.
- If the Google Maps Reviews tab appears empty in the accessibility tree, click the 'All reviews' radio button to focus the reviews container and press the 'PageDown' key to trigger the asynchronous rendering of individual review cards.
- The GraphQL `CreateReview` mutation expects the `publishedDate` variable to be a valid ISO 8601 Timestamp string. Relative dates (e.g., "3 months ago") must be omitted or normalized to prevent GraphQL validation errors.
- Official business websites can embed an Instagram profile badge showing active follower and post counts, which is a highly reliable fallback for extracting Instagram follower numbers without navigating to Instagram directly.
- When Google Maps has two active pins for a rebranded venue (e.g., 'CJs Dining (Formerly Potato Corner Craigieburn)' alongside legacy 'Potato Corner Craigieburn'), treat the successor as a new listing with its own Place ID/sourceUrl, since the UpdateListingData mutation does not accept name updates.
- A Google Maps listing marked 'Temporarily closed' may actually be open; cross-reference with active posts or hours on their official Facebook page, as Facebook remains the most active channel for real-time operational updates.
- A Google Maps visitor update claiming a venue is closed to the public (e.g., 'wholesale orders only') does not mean the business is permanently closed; cross-reference with ABN/ACN status and their website/socials to verify if they are active as a wholesale/online-only business.
- Distinct businesses sharing a physical venue (e.g. stalls inside Footscray Market) must not share the same `sourceUrl` (such as the market's generic Maps URL) to avoid triggering exact-URL duplicate matches and incorrect merges in the push script; use unique profile, website, or specific trader paths instead.
- When clicking suggestions or related places within a Google Maps details panel, the resulting URL contains coordinate chains for both the query and target listings; parse_lat_lng_from_url will return the first occurrence (the query/previous listing), so check the last occurrence of !3d/!4d parameters to resolve the correct target location.
- When multiple Filipino doctors/dentists practice at the same general clinic, register them as separate listings named 'Dr. <Name> - <Clinic Name>' using the shared clinic details to avoid duplicate collisions.
